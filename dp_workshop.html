The DP Workshop

All 21 NeetCode DP Problems — 5 Templates

The Workshop Map

The Workshop is a vast crafting hall where five master artisans each work at their own station. Each has their own tools, their own ritual, and their own way of turning raw material into a finished answer. Every DP problem belongs to one of these five stations.

The Workshop Map — What Everything Means

Workshop Table = Array/String
The raw material sitting on the table. Could be a row of numbers, a string of characters, or a list of prices. The craftsperson walks along it, step by step.

In code: nums[], s, prices[]
Chalk/Notebook = Memoized Results
As the craftsperson works, they write results in chalk on the table or in a notebook. Later steps look back at these notes instead of recalculating from scratch.

In code: dp[] or dp[][]
Looking Back = Previous Results
The craftsperson never starts from scratch. They look back at what they already computed — the last step, the last two steps, or a specific earlier position.

In code: dp[i-1], dp[i-2], dp[i][j-1]
Base Case = Starting Values
Before the loop begins, the first few entries are set by hand. These are the golden base cases — the foundation everything else builds on.

In code: dp[0] = ..., dp[0][0] = True

The 5 Scenes

Each scene is a ritual. Each step in the ritual maps to exactly one line of code. Memorize the ritual, you've memorized the algorithm.

Scene 1: Maya's Staircase
Linear DP — step up, look back 1-2 steps, combine
THE RITUAL
❶ Maya stands at the bottom. Writes GOLDEN BASE CASES on the first 1-2 steps. dp[0], dp[1] = base
❷ Climbs to step i. LOOKS BACK at steps i-1 and i-2.
❸ COMBINES their chalk values using the formula. Writes the new value on step i.
❹ Repeats for every step until the top.
❺ Reads the chalk on the TOP STEP. That's the answer.
RITUAL → CODE
❶
Golden base cases on first steps.
dp[0], dp[1] = base_cases
❷❸
Look back, combine — write on step i.
dp[i] = combine(dp[i-1], dp[i-2])
❹
Repeat for every step.
for i in range(2, n):
❺
Top step = the answer.
return dp[n-1]
Common Mistakes
WRONG — forgetting base cases
for i in range(2, n):
    dp[i] = dp[i-1] + dp[i-2]
# dp[0] and dp[1] never set!
# Everything builds on garbage.
RIGHT — always set base cases first
dp[0], dp[1] = 1, 2  # GOLDEN
for i in range(2, n):
    dp[i] = dp[i-1] + dp[i-2]
WRONG — reading dp[i-2] when i < 2
for i in range(0, n):
    dp[i] = dp[i-1] + dp[i-2]
# IndexError at i=0 and i=1!
RIGHT — start loop at i = 2
for i in range(2, n):  # safe!
    dp[i] = dp[i-1] + dp[i-2]
# i-2 is always >= 0
Scene 2: Forge's Anvil
Target/Knapsack DP — fill shelf slots with molds
THE RITUAL
❶ Forge sets up a SHELF with slots 0 to amount. Slot 0 gets a GOLDEN ZERO. Rest get ∞ (or 0 for counting).
❷ Picks up the FIRST MOLD (coin). Walks the shelf left to right.
❸ At each slot a: Can this mold fit? Check slot a-coin. HAMMER: dp[a] = min(dp[a], dp[a-coin]+1).
❹ Picks the NEXT MOLD. Repeats the full shelf sweep.
❺ Reads slot[amount]. That's the answer. Still ∞? Impossible.
Unbounded vs 0-1: Unbounded (can reuse coins) = inner loop LEFT to RIGHT. 0-1 Knapsack (each item once) = inner loop RIGHT to LEFT. Direction matters!
RITUAL → CODE
❶
Shelf setup. Slot 0 = golden zero.
dp = [inf]*(amount+1); dp[0] = 0
❷
Pick up mold (outer loop = coins).
for coin in coins:
❸
Sweep shelf. Hammer each slot.
for a in range(coin, amount+1): dp[a] = min(dp[a], dp[a-coin]+1)
❹❺
All molds done. Read slot[amount].
return dp[amount] if dp[amount] != inf else -1
Common Mistakes
WRONG — inner loop starts at 0
for a in range(0, amount + 1):
    dp[a] = min(dp[a], dp[a - coin] + 1)
# dp[a-coin] with negative index!
RIGHT — inner loop starts at coin
for a in range(coin, amount + 1):
    dp[a] = min(dp[a], dp[a - coin] + 1)
# a-coin is always >= 0
WRONG — wrong loop direction for 0-1 knapsack
# Target Sum: each num used ONCE
for a in range(num, goal + 1):  # LEFT to RIGHT
    dp[a] += dp[a - num]
# Same num counted multiple times!
RIGHT — right to left for 0-1
# 0-1 knapsack: RIGHT to LEFT
for a in range(goal, num - 1, -1):
    dp[a] += dp[a - num]
# Each num only used once!
Scene 3: Echo's Mirrors
Palindrome DP — expand from center, check reflections
THE RITUAL
❶ Echo stands at the first tile. ODD STANCE — stands ON the center tile. l = r = center.
❷ Checks s[l] == s[r]. MATCH? GREEN GLOW. Count it! Expand: l--, r++.
❸ Keeps expanding until MISMATCH (RED FLASH) or out of bounds. Stops.
❹ Returns to center. EVEN STANCE — stands BETWEEN two tiles. l = center, r = center+1.
❺ Same expansion. Done with this center? Walks to the next tile. Repeat.
RITUAL → CODE
❶
Odd stance — center on one tile.
l, r = center, center
❷❸
Expand while match. Count each palindrome.
while l>=0 and r<len(s) and s[l]==s[r]: count+=1; l-=1; r+=1
❹
Even stance — between two tiles.
l, r = center, center + 1
❺
Next center. Repeat for all tiles.
for center in range(len(s)):
Common Mistakes
WRONG — only checking odd-length
l, r = center, center
while s[l] == s[r]: ...
# Misses "aa", "abba" — even palindromes!
RIGHT — check BOTH odd and even
# Odd: (center, center)
# Even: (center, center + 1)
for l, r in [(c, c), (c, c+1)]:
    while ... expand ...
WRONG — using O(n^2) dp table
dp = [[False]*n for _ in range(n)]
# O(n^2) space! Unnecessary for
# palindrome counting/finding.
RIGHT — expand from center = O(1) space
# O(1) space, O(n^2) time
# Just two pointers expanding out
l, r = center, center
while ... expand ...
Scene 4: Nila's Loom
2D Table DP — match=diagonal, else=max(up, left)
THE RITUAL
❶ Nila sets up the LOOM. Row headers = text1 chars. Column headers = text2 chars. First row and column all ZEROS (white thread).
❷ Moves to cell (i,j). Checks: does text1[i-1] MATCH text2[j-1]?
❸ MATCH! Pulls GOLDEN THREAD from DIAGONAL (i-1,j-1). Ties knot = diagonal value + 1.
❹ NO MATCH. Checks UP cell (i-1,j) and LEFT cell (i,j-1). Pulls SILVER THREAD from the LARGER one.
❺ Fills every cell, row by row. Bottom-right cell = the answer. GOLDEN GLOW.
RITUAL → CODE
❶
Set up loom. First row/col = zeros.
dp = [[0]*(n+1) for _ in range(m+1)]
❷
Check match at cell (i,j).
if text1[i-1] == text2[j-1]:
❸
Match! Golden thread from diagonal + 1.
dp[i][j] = dp[i-1][j-1] + 1
❹
No match. Silver thread from max(up, left).
dp[i][j] = max(dp[i-1][j], dp[i][j-1])
❺
Bottom-right = answer.
return dp[m][n]
Common Mistakes
WRONG — off-by-one indexing
if text1[i] == text2[j]:
    dp[i][j] = dp[i-1][j-1] + 1
# Grid is 1-indexed, strings are 0-indexed!
# text1[i] is wrong when grid starts at 1.
RIGHT — use i-1 for string index
if text1[i-1] == text2[j-1]:
    dp[i][j] = dp[i-1][j-1] + 1
# Grid row i maps to string index i-1
WRONG — forgetting +1 in grid size
dp = [[0]*n for _ in range(m)]
# No room for base case row/column!
# dp[0][j] and dp[i][0] would be data, not base.
RIGHT — (m+1) x (n+1) grid
dp = [[0]*(n+1) for _ in range(m+1)]
# Row 0 and Col 0 = base cases (all 0)
# Actual data starts at dp[1][1]
Scene 5: Ace's Table
State Machine / Interval DP — multiple state cards, simultaneous swap
THE RITUAL
❶ Ace lays THREE STATE CARDS on the table: RED (hold), GREEN (sold), BLUE (rest). Sets starting chips.
❷ Day advances. Price ticker updates. Ace calculates new values for ALL THREE cards simultaneously.
❸ HOLD = max(keep holding, buy today from rest). SOLD = hold + today's price. REST = max(keep resting, done cooling from sold).
❹ Swaps all three cards AT ONCE (temp variables). Repeats for each day.
❺ Game over. Answer = max(sold, rest). Can't end while holding!
Why simultaneous swap: If you update hold first and then use it to compute sold, you're using today's hold decision to sell today — that's time travel. Compute ALL new values first, THEN swap.
RITUAL → CODE
❶
Lay state cards. Set starting chips.
hold, sold, rest = -prices[0], 0, 0
❷❸
Compute new values for all cards.
new_hold=max(hold,rest-price); new_sold=hold+price; new_rest=max(rest,sold)
❹
Swap all at once.
hold, sold, rest = new_hold, new_sold, new_rest
❺
Game over. Answer = max(sold, rest).
return max(sold, rest)
Common Mistakes
WRONG — updating states sequentially
hold = max(hold, rest - price)
sold = hold + price  # USING UPDATED HOLD!
rest = max(rest, sold)
# Buy and sell same day = time travel!
RIGHT — compute all, then swap
new_hold = max(hold, rest - price)
new_sold = hold + price
new_rest = max(rest, sold)
hold, sold, rest = new_hold, new_sold, new_rest
WRONG — returning max(hold, sold, rest)
return max(hold, sold, rest)
# Can't end the game while holding stock!
# hold might be positive but unrealized.
RIGHT — only sold or rest
return max(sold, rest)
# You must have SOLD your stock or be
# RESTING (already sold before). No holding!

21 Problems by Scene

Scene 1 — Maya's Staircase (Linear DP)

1 Climbing Stairs (#70) Easy

▶
PROBLEM
You are climbing a staircase. It takes n steps to reach the top. Each time you can climb 1 or 2 steps. In how many distinct ways can you climb to the top?
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Target/coins? No. Multiple states? No. → Scene 1 Maya's Staircase. Pure linear recurrence.
THE RITUAL
dp[i] = dp[i-1] + dp[i-2]. Ways to reach step i = ways from i-1 (one step) + ways from i-2 (two steps). Pure Fibonacci.
WHAT CHANGES FROM THE TEMPLATE
Space-optimized to two variables. No array needed — only ever look back 1-2 steps.
def climbStairs(n):
    if n <= 2: return n
    prev2, prev1 = 1, 2
    for i in range(3, n + 1):
        prev2, prev1 = prev1, prev1 + prev2
    return prev1

2 Min Cost Climbing Stairs (#746) Easy

▶
PROBLEM
Given an array cost where cost[i] is the cost of step i, find the minimum cost to reach the top. You can start from step 0 or step 1, and climb 1 or 2 steps at a time.
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Target? No. Multiple states? No. → Scene 1 Linear recurrence with min.
THE RITUAL
dp[i] = min(dp[i-1]+cost[i-1], dp[i-2]+cost[i-2]). Cheapest way to reach step i. Cost comes from the step you jump FROM.
WHAT CHANGES FROM THE TEMPLATE
min instead of sum. Costs come from the departure step, not arrival.
def minCostClimbingStairs(cost):
    prev2, prev1 = 0, 0
    for i in range(2, len(cost) + 1):
        prev2, prev1 = prev1, min(prev1 + cost[i-1], prev2 + cost[i-2])
    return prev1

3 House Robber (#198) Medium

▶
PROBLEM
Given an array of non-negative integers representing money in each house, determine the maximum amount you can rob without robbing two adjacent houses.
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Target? No. Multiple states? No. → Scene 1 Take-or-skip linear DP.
THE RITUAL
dp[i] = max(dp[i-1], dp[i-2]+nums[i]). Rob this house (skip prev, take i) or skip this house (keep prev). Classic take-or-skip.
WHAT CHANGES FROM THE TEMPLATE
max instead of sum. Decision at each step: take or skip.
def rob(nums):
    if len(nums) <= 2: return max(nums)
    prev2, prev1 = nums[0], max(nums[0], nums[1])
    for i in range(2, len(nums)):
        prev2, prev1 = prev1, max(prev1, prev2 + nums[i])
    return prev1

4 House Robber II (#213) Medium

▶
PROBLEM
Same as House Robber, but houses are arranged in a circle. First and last houses are adjacent.
FLOWCHART REASONING
Same as House Robber but circular constraint. Still Scene 1 — just run it twice.
THE RITUAL
Circular! Can't rob both first and last. Run House Robber twice: once skipping first, once skipping last. Answer = max of both.
WHAT CHANGES FROM THE TEMPLATE
Two passes of the same linear DP on different slices. Answer = max of both passes.
def rob(nums):
    if len(nums) <= 2: return max(nums)
    def rob_line(arr):
        prev2, prev1 = arr[0], max(arr[0], arr[1])
        for i in range(2, len(arr)):
            prev2, prev1 = prev1, max(prev1, prev2 + arr[i])
        return prev1
    return max(rob_line(nums[1:]), rob_line(nums[:-1]))

5 Decode Ways (#91) Medium

▶
PROBLEM
Given a string of digits, determine the total number of ways to decode it. 'A'=1, 'B'=2, ..., 'Z'=26.
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Target? No. Multiple states? No. → Scene 1 Conditional lookback.
THE RITUAL
dp[i] depends on dp[i-1] (single digit valid?) and dp[i-2] (two-digit valid?). Not always looking back 2 — only if the two-digit number is 10-26.
WHAT CHANGES FROM THE TEMPLATE
Conditional lookback. Single digit '0' contributes nothing. Two-digit must be 10-26. Watch for '0' edge cases.
def numDecodings(s):
    if s[0] == '0': return 0
    prev2, prev1 = 1, 1
    for i in range(1, len(s)):
        curr = 0
        if s[i] != '0':
            curr += prev1
        if 10 <= int(s[i-1:i+1]) <= 26:
            curr += prev2
        prev2, prev1 = prev1, curr
    return prev1

6 Maximum Product Subarray (#152) Medium

▶
PROBLEM
Given an integer array, find a contiguous subarray that has the largest product, and return its product.
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Target? No. Multiple states? Kind of — but it's still linear with two tracking values. → Scene 1 Linear DP with dual tracking.
THE RITUAL
Track BOTH max and min at each step (negative x negative = positive). dp_max[i] and dp_min[i] evolve together.
WHAT CHANGES FROM THE TEMPLATE
Two values per step instead of one. Need min because a negative number can flip min to max.
def maxProduct(nums):
    res = max(nums)
    cur_max, cur_min = 1, 1
    for n in nums:
        vals = (n, n * cur_max, n * cur_min)
        cur_max, cur_min = max(vals), min(vals)
        res = max(res, cur_max)
    return res

Scene 2 — Forge's Anvil (Knapsack/Target DP)

7 Coin Change (#322) Medium

▶
PROBLEM
Given coins of different denominations and a total amount, return the fewest number of coins needed. Return -1 if impossible.
FLOWCHART REASONING
Two sequences? No. Palindrome? No. Building to a target amount using coins? YES! → Scene 2 Forge's Anvil.
THE RITUAL
Pure template. Min coins to reach amount. Unbounded (can reuse coins) = left to right inner loop.
WHAT CHANGES FROM THE TEMPLATE
Nothing — this IS the template. Minimum count. Unbounded knapsack.
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for a in range(coin, amount + 1):
            dp[a] = min(dp[a], dp[a - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

8 Coin Change II (#518) Medium

▶
PROBLEM
Given coins and an amount, return the number of combinations that make up that amount.
FLOWCHART REASONING
"How many ways to make amount" using coins. → Scene 2 Forge's Anvil. Count combinations.
THE RITUAL
COUNT combinations (not min). dp[0] = 1 (one way to make 0). Outer loop = coins ensures unique combos (no duplicates like [1,2] and [2,1]).
WHAT CHANGES FROM THE TEMPLATE
+= instead of min. Base case = 1 instead of 0. Init rest to 0 (not inf).
def change(amount, coins):
    dp = [0] * (amount + 1)
    dp[0] = 1
    for coin in coins:
        for a in range(coin, amount + 1):
            dp[a] += dp[a - coin]
    return dp[amount]

9 Target Sum (#494) Medium

▶
PROBLEM
Given an array of integers and a target, assign + or - to each integer to make the sum equal target. Return the number of ways.
FLOWCHART REASONING
"How many ways to reach target sum" with items. → Scene 2 Forge's Anvil. 0-1 knapsack variant.
THE RITUAL
Transform to subset sum. Each num used ONCE = loop RIGHT TO LEFT (0-1 knapsack). Math trick: target = P - (total - P) means P = (total + target) / 2.
WHAT CHANGES FROM THE TEMPLATE
Reverse inner loop direction (0-1 knapsack). Mathematical transformation to reduce to subset sum.
def findTargetSumWays(nums, target):
    total = sum(nums)
    if (total + target) % 2 or abs(target) > total: return 0
    goal = (total + target) // 2
    dp = [0] * (goal + 1)
    dp[0] = 1
    for num in nums:
        for a in range(goal, num - 1, -1):
            dp[a] += dp[a - num]
    return dp[goal]

10 Word Break (#139) Medium

▶
PROBLEM
Given a string s and a dictionary of words, determine if s can be segmented into a space-separated sequence of dictionary words.
FLOWCHART REASONING
"Can you break s into dictionary words?" = build s from word pieces (like building an amount from coins). → Scene 2 Forge's Anvil.
THE RITUAL
dp[i] = can s[:i] be segmented? Try every split point j: if dp[j] is True AND s[j:i] is a word, then dp[i] = True.
WHAT CHANGES FROM THE TEMPLATE
Boolean dp. Inner loop tries all split points instead of fixed coin sizes. Words replace coins, string length replaces amount.
def wordBreak(s, wordDict):
    dp = [False] * (len(s) + 1)
    dp[0] = True
    words = set(wordDict)
    for i in range(1, len(s) + 1):
        for j in range(i):
            if dp[j] and s[j:i] in words:
                dp[i] = True
                break
    return dp[len(s)]

Scene 3 — Echo's Mirrors (Palindrome DP)

11 Longest Palindromic Substring (#5) Medium

▶
PROBLEM
Given a string s, return the longest palindromic substring in s.
FLOWCHART REASONING
"Palindromic substring" — palindrome keyword! → Scene 3 Echo's Mirrors.
THE RITUAL
Expand from center. Track the longest palindrome found. Store the actual substring, not just count.
WHAT CHANGES FROM THE TEMPLATE
Instead of counting, track the longest substring found so far. Update result when a longer palindrome is found.
def longestPalindrome(s):
    res = ""
    for center in range(len(s)):
        for l, r in [(center, center), (center, center + 1)]:
            while l >= 0 and r < len(s) and s[l] == s[r]:
                if r - l + 1 > len(res):
                    res = s[l:r+1]
                l -= 1; r += 1
    return res

12 Palindromic Substrings (#647) Medium

▶
PROBLEM
Given a string s, return the number of palindromic substrings in it.
FLOWCHART REASONING
"Palindromic substrings" — palindrome keyword! → Scene 3 Echo's Mirrors.
THE RITUAL
Pure template. Count every palindrome found during expansion. Each successful match in the while loop = one more palindromic substring.
WHAT CHANGES FROM THE TEMPLATE
Nothing — this IS the template. Pure counting.
def countSubstrings(s):
    count = 0
    for center in range(len(s)):
        for l, r in [(center, center), (center, center + 1)]:
            while l >= 0 and r < len(s) and s[l] == s[r]:
                count += 1
                l -= 1; r += 1
    return count

Scene 4 — Nila's Loom (2D Table DP)

13 Unique Paths (#62) Medium

▶
PROBLEM
A robot is on an m x n grid. It can only move right or down. How many unique paths exist from top-left to bottom-right?
FLOWCHART REASONING
Two dimensions (grid), paths through a 2D space. → Scene 4 Nila's Loom. Simplified 2D table.
THE RITUAL
Only DOWN and RIGHT. dp[i][j] = up + left. First row and column all 1s (only one way to reach any edge cell).
WHAT CHANGES FROM THE TEMPLATE
No diagonal. No match/mismatch. Simpler than LCS — always sum up + left. Base = all 1s on edges.
def uniquePaths(m, n):
    dp = [[1] * n for _ in range(m)]
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
    return dp[m-1][n-1]

14 Longest Common Subsequence (#1143) Medium

▶
PROBLEM
Given two strings text1 and text2, return the length of their longest common subsequence.
FLOWCHART REASONING
"Longest common" between two strings — two sequences being compared. → Scene 4 Nila's Loom.
THE RITUAL
Pure template. Match = diagonal + 1. No match = max(up, left). This IS the canonical 2D table problem.
WHAT CHANGES FROM THE TEMPLATE
Nothing — this IS the template.
def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

15 Edit Distance (#72) Medium

▶
PROBLEM
Given two strings word1 and word2, return the minimum number of operations (insert, delete, replace) to convert word1 to word2.
FLOWCHART REASONING
"Edit distance" between two strings — comparing two sequences. → Scene 4 Nila's Loom.
THE RITUAL
Match = diagonal (free, no cost). No match = 1 + min(diagonal=replace, up=delete, left=insert). Three-way min instead of two-way max.
WHAT CHANGES FROM THE TEMPLATE
Base cases = distance to empty string (0,1,2,...). Three neighbors instead of two. min instead of max. Match = free (no +1 needed).
def minDistance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = i
    for j in range(n + 1): dp[0][j] = j
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1])
    return dp[m][n]

16 Interleaving String (#97) Medium

▶
PROBLEM
Given strings s1, s2, and s3, determine if s3 is formed by interleaving s1 and s2.
FLOWCHART REASONING
"Interleaving" two strings to form a third — comparing sequences. → Scene 4 Nila's Loom.
THE RITUAL
dp[i][j] = can s3[:i+j] be formed from s1[:i] and s2[:j]? Check if next char in s3 matches s1 (from up) or s2 (from left).
WHAT CHANGES FROM THE TEMPLATE
Boolean grid. Three strings involved but grid is still 2D (s1 x s2). Check s3[i+j-1] against s1[i-1] or s2[j-1].
def isInterleave(s1, s2, s3):
    m, n = len(s1), len(s2)
    if m + n != len(s3): return False
    dp = [[False] * (n + 1) for _ in range(m + 1)]
    dp[0][0] = True
    for i in range(1, m + 1):
        dp[i][0] = dp[i-1][0] and s1[i-1] == s3[i-1]
    for j in range(1, n + 1):
        dp[0][j] = dp[0][j-1] and s2[j-1] == s3[j-1]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            dp[i][j] = (dp[i-1][j] and s1[i-1] == s3[i+j-1]) or \
                        (dp[i][j-1] and s2[j-1] == s3[i+j-1])
    return dp[m][n]

17 Distinct Subsequences (#115) Hard

▶
PROBLEM
Given strings s and t, return the number of distinct subsequences of s which equals t.
FLOWCHART REASONING
"Distinct subsequences" — comparing s against t. Two sequences. → Scene 4 Nila's Loom.
THE RITUAL
dp[i][j] = ways to form t[:j] from s[:i]. Always inherit from UP (skip char in s). If match, also ADD diagonal value.
WHAT CHANGES FROM THE TEMPLATE
+= diagonal on match instead of choosing max. Always inherit UP value. Base case: dp[i][0] = 1 (empty t always matches).
def numDistinct(s, t):
    m, n = len(s), len(t)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = 1
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            dp[i][j] = dp[i-1][j]
            if s[i-1] == t[j-1]:
                dp[i][j] += dp[i-1][j-1]
    return dp[m][n]

18 Regular Expression Matching (#10) Hard

▶
PROBLEM
Given a string s and a pattern p with '.' (any char) and '*' (zero or more of preceding), determine if the pattern matches the entire string.
FLOWCHART REASONING
"Regex matching" — comparing string against pattern. Two sequences. → Scene 4 Nila's Loom.
THE RITUAL
Boolean 2D grid. '.' matches any char (diagonal). '*' means zero-or-more of preceding: zero = dp[i][j-2], more = dp[i-1][j] if char matches.
WHAT CHANGES FROM THE TEMPLATE
Special handling for '*' (look 2 cols back for zero matches, or inherit from UP for one-more match). '.' acts as wildcard.
def isMatch(s, p):
    m, n = len(s), len(p)
    dp = [[False] * (n + 1) for _ in range(m + 1)]
    dp[0][0] = True
    for j in range(1, n + 1):
        if p[j-1] == '*':
            dp[0][j] = dp[0][j-2]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if p[j-1] == '*':
                dp[i][j] = dp[i][j-2]
                if p[j-2] == '.' or p[j-2] == s[i-1]:
                    dp[i][j] = dp[i][j] or dp[i-1][j]
            elif p[j-1] == '.' or p[j-1] == s[i-1]:
                dp[i][j] = dp[i-1][j-1]
    return dp[m][n]

19 Best Time to Buy & Sell Stock with Cooldown (#309) Medium

▶
PROBLEM
Given an array of stock prices, find the maximum profit. After selling, you must wait one day before buying again (cooldown).
FLOWCHART REASONING
"Buy and sell stock" with "cooldown" — multiple states per step. → Scene 5 Ace's Table.
THE RITUAL
Pure template. Three state cards: hold (bought, waiting to sell), sold (just sold today), rest (cooling down or idle). Simultaneous swap.
WHAT CHANGES FROM THE TEMPLATE
Nothing — this IS the template.
def maxProfit(prices):
    hold, sold, rest = -prices[0], 0, 0
    for price in prices[1:]:
        new_hold = max(hold, rest - price)
        new_sold = hold + price
        new_rest = max(rest, sold)
        hold, sold, rest = new_hold, new_sold, new_rest
    return max(sold, rest)

20 Longest Increasing Subsequence (#300) Medium

▶
PROBLEM
Given an integer array nums, return the length of the longest strictly increasing subsequence.
FLOWCHART REASONING
"Increasing subsequence" — at each position you track a state (best LIS ending here) and look back at ALL previous. → Scene 5 Ace's Table.
THE RITUAL
dp[i] = length of LIS ending at i. For each i, check ALL previous j where nums[j] < nums[i]. Answer = max of entire dp array.
WHAT CHANGES FROM THE TEMPLATE
O(n^2). For each position, look back at ALL previous positions (not just i-1, i-2). Answer = max of entire dp array, not just the last element.
def lengthOfLIS(nums):
    dp = [1] * len(nums)
    for i in range(1, len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

21 Burst Balloons (#312) Hard

▶
PROBLEM
Given n balloons with values, burst them to maximize coins. Bursting balloon i gives nums[left] * nums[i] * nums[right]. Return maximum coins.
FLOWCHART REASONING
"Burst balloons" — interval problem, processing ranges. Multiple states, choosing the LAST element in an interval. → Scene 5 Ace's Table (Interval DP).
THE RITUAL
Interval DP. dp[l][r] = max coins from popping all balloons between l and r. Choose the LAST balloon to pop (k). Boundaries l and r are always there (padding with 1s).
WHAT CHANGES FROM THE TEMPLATE
Triple nested loop. Iterate by interval LENGTH (small to large). Think backwards — which balloon pops LAST. Pad array with 1s at boundaries.
def maxCoins(nums):
    nums = [1] + nums + [1]
    n = len(nums)
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n):
        for l in range(n - length):
            r = l + length
            for k in range(l + 1, r):
                dp[l][r] = max(dp[l][r],
                    dp[l][k] + dp[k][r] + nums[l] * nums[k] * nums[r])
    return dp[0][n-1]

1. What scene for: "How many ways to climb n stairs taking 1 or 2 steps at a time?"

4. What scene for: "Find the longest common subsequence of two strings?"
Scene 4 Nila's Loom (2D Table DP). Two sequences being compared. Match = diagonal + 1. No match = max(up, left).
5. What scene for: "Best time to buy and sell stock with cooldown?"
Scene 5 Ace's Table (State Machine DP). Three state cards: hold, sold, rest. Compute all new values simultaneously, then swap.
6. In Linear DP (Scene 1), why can you optimize to O(1) space?
Only need last 1-2 values, not the full array. Since dp[i] only depends on dp[i-1] and dp[i-2], you can replace the entire array with just two variables: prev1 and prev2. Everything older is never accessed again.
7. In Coin Change (Scene 2), why does the inner loop go LEFT to RIGHT?
Unbounded: same coin can be reused. Going left to right means when you compute dp[a], dp[a-coin] may already include this coin — which is correct because you CAN reuse it. For 0-1 knapsack (Target Sum), loop RIGHT to LEFT so each item is only used once.
8. In the 2D Table (Scene 4), what do the three neighbors represent?
Diagonal = both chars consumed (match). Up = skip char from text1 (delete). Left = skip char from text2 (insert). These three directions cover all possible operations when comparing two sequences.
9. In Buy & Sell Cooldown (Scene 5), why must you swap all states simultaneously?
Using an already-updated value within the same day would let you buy and sell on the same day. If you update hold first (buying today), then immediately use the new hold to compute sold (selling today), you've time-traveled. Compute ALL new values from yesterday's cards, THEN replace.
10. What makes Burst Balloons an "interval DP" instead of a regular 2D table?
dp[l][r] represents a range/interval, not two sequence positions. You iterate by interval LENGTH (small intervals first, building up to larger ones) and at each interval you choose the LAST element to process. This "think backwards" approach is the hallmark of interval DP.
